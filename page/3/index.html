浏览器缓存问题

发表于 2018-03-29 更新于 2020-12-05 分类于 HTML5 Changyan：

请求缓存的路线图:

两个概念

  1. 强缓存
    用户发送的请求，直接从客户端缓存中获取，不发送请求到服务器，不与服务器发生交互行为。

  2. 协商缓存
    用户发送的请求，发送到服务器后，由服务器判定是否从缓存中获取资源。

共同点: 客户端获得的数据最后都是从客户端缓存中获得

区别: 强缓存不与服务器交互，而协商缓存则需要与服务器交互。
